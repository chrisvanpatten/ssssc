Sexy Style Sheets with Sass

(and Compass)

What's the problem with CSS?

  • Static

    No programmatic capabilities

  • Repetition

    Constantly repeating key colors, values, bits of style

  • Vendor prefixes

    New features are prefixed, and often have varying syntaxes

  • Just to name a few...

How can Sass help?

  • Sass (and other preprocessors) extend and improve CSS

    They're technically "meta-languages"

  • Sass provides programmatic capabilities

    Variables, math, functions, basic scripting

  • Sass introduces new syntax

    Stop repeating yourself with nesting and extending

  • Sass compiles to CSS

    It replaces CSS in your workflow, but is compiled to CSS for production

Variables

Input...

$blue: #3bbfce;
$margin: 16px;
$sans-serif: 'Helvetica Neue', 'Helvetica', 'Arial', sans-serif;

div {
	color: $blue;
	font: 16px 'Museo Sans', $sans-serif;
	margin: $margin;
}

Output...

div {
  color: #3bbfce;
  font: 16px "Museo Sans", "Helvetica Neue", "Helvetica", "Arial", sans-serif;
  margin: 16px;
}

Math

Math

div {
	width: 300px - (8px + 8px);
	padding: 8px;
	margin: $margin;
}

Math with variables

$margin: 16px;

div {
	width: 300px - $margin;
	padding: $margin / 2;
	margin: $margin;
}

Math

Output...

div {
  width: 284px;
  padding: 8px;
  margin: 16px;
}

Nesting

Input...

table.hl {
	margin: 2em 0;

	td.ln {
		text-align: right;
	}
}

li {
	font: {
		family: serif;
		weight: bold;
		size: 1.2em;
	}
}

Nesting

Output...

table.hl {
  margin: 2em 0;
}
table.hl td.ln {
  text-align: right;
}

li {
  font-family: serif;
  font-weight: bold;
  font-size: 1.2em;
}

Nesting

Input...

$link-color: #0000FF;

a {
	color: $link-color;

	&:hover, &:visited:hover {
		color: darken($link-color, 10%);
	}

	&:visited {
		color: darken($link-color, 5%);
	}

	&:active {
		color: lighten($link-color, 10%);
	}
}

Nesting

Output...

a {
  color: blue;
}
a:hover, a:visited:hover {
  color: #0000cc;
}
a:visited {
  color: #0000e6;
}
a:active {
  color: #3333ff;
}

Mixins

Input...

@mixin opacity($amount) {
	filter: progid:DXImageTransform.Microsoft.
Alpha(Opacity=#{$amount * 100});
	opacity: $amount;
}

a {
	@include opacity(0.2);
}

Output...

a {
  filter: progid:DXImageTransform.Microsoft.
Alpha(Opacity=20);
  opacity: 0.2;
}

Mixins

Input...

@mixin image-replacement() {
	text-indent: -9999em;
	overflow: hidden;
}

a {
	@include image-replacement();
}

Output...

a {
	text-indent -9999em;
	overflow: hidden;
}

Extending and inheritance

Input...

.box {
	border: 2px solid black;
	margin: 1em;
}

.important-box {
	@extends .box;
	background-color: #CCC;
	padding-left: 48px;
}

.error-box {
	@extends .important-box;
	background-color: #FFF;
}

Extending and inheritance

Output...

.box, .important-box, .error-box {
	border: 2px solid black;
	margin: 1em;
}

.important-box, .error-box {
	background-color: #CCC;
	padding-left: 48px;
}

.error-box {
	background-color: #FCC;
}

So why Sass? Why not Less/Stylus/etc?

  • CSS3 is borrowing from Sass

    See nesting spec

  • Smarter, more extensible syntax

  • Compass.

    Compass will blow your mind.

Compass

Compass is a framework built on Sass, offering curated mixins and additional functionality.

Like what?

  • A full range of pre-built CSS3 mixins

  • Intelligent URL handlers for making asset management a breeze

  • Callbacks so you can do "stuff" on compilation

  • Automatic image generation (sprites and more)

  • and that's just the start!
